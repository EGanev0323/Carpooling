# Функционална спецификация

## 1. Визия

Уеб платформа за карпулинг между градове в България. Шофьор публикува
маршрут (тръгва от град А в час X, свободни N места, цена P лв./човек);
пътници търсят по origin/destination и дата, резервират място, комуникират
с шофьора в чат и след пътуването се оценяват взаимно.

## 2. Потребителски роли

| Роля | Описание |
|---|---|
| **Гост** | Разглежда обявените маршрути, вижда карта, не може да резервира. |
| **Потребител** | Логнат акаунт; може да бъде шофьор и/или пътник. |
| **Шофьор** (роля на потребител) | Има верифициран профил на автомобил, може да публикува маршрути. |
| **Администратор** | Достъп до аудит лога, може да блокира потребители, вижда статистика. |

Един акаунт може да бъде едновременно и шофьор, и пътник. Ролята „шофьор“
се активира чрез отделен flow за добавяне на автомобил.

## 3. Основни use cases

### UC-1: Регистрация
- Формуляр: имейл, парола (мин. 12 знака, zxcvbn score ≥ 3), собствено име,
  фамилия, телефон (валидиран по формат `+359...` или `08...`).
- Cloudflare Turnstile challenge.
- Имейл потвърждение с еднократен линк (валиден 24h).
- След потвърждение — автоматичен login и redirect към профил.

### UC-2: Логин
- Email + парола + Turnstile challenge.
- 5 неуспешни опита за 15 минути → temporary lock (429).
- Успешен login → HttpOnly JWT cookie + refresh token cookie.

### UC-3: Добавяне на автомобил (активиране на шофьор)
- Марка, модел, година, регистрационен номер (маскиран в UI за други),
  цвят, брой места (2–8), удобства (климатик, животни, багаж, музика).
- Задължителна снимка на автомобила.
- След добавяне потребителят може да публикува маршрути.

### UC-4: Публикуване на маршрут
- Origin (град от dropdown с автокомплийт), Destination.
- Дата и час на тръгване.
- Свободни места (1 до `car.seats - 1`).
- Цена на човек (1–200 лв.).
- Опционално: междинни спирки, коментар („не пуша“, „без животни“ и т.н.).
- Автоматично изчисление на разстояние и препоръчителна цена (базирано на
  среден разход 12 лв./100 км ÷ брой места, като референс).

### UC-5: Търсене на маршрут
- Филтри: origin, destination, дата (± 1 ден), min. свободни места,
  удобства.
- Сортиране: по време на тръгване (default), цена, рейтинг на шофьор.
- Резултати: списък + карта (Leaflet) с полилинии на всеки маршрут.
- Клик на резултат → детайлна страница с профил на шофьор, рейтинг, чат-бутон.

### UC-6: Резервация
- Пътникът избира брой места (1 до наличните).
- Оптимистично заключване — ако между показване и резервация местата се
  запълнят, показва се грешка и бутон „опресни“.
- Статус на резервация: `PENDING` → `CONFIRMED` (от шофьор) / `REJECTED`
  / `CANCELLED` (от пътник до 12h преди тръгване без такса).

### UC-7: Real-time чат
- Между шофьор и всеки от пътниците с потвърдена резервация.
- 1-към-1 канали (не групов чат в MVP).
- WebSocket (STOMP) с fallback към polling всеки 3s ако WS падне.
- Съобщенията се пазят 90 дни, после автоматично се пращат в архивна таблица.
- Максимум 500 знака на съобщение.
- Спам защита: 10 съобщения/минута/канал.

### UC-8: Рейтинг и отзив
- Достъпен 2 часа след планирания час на пристигане.
- Пътникът оценява шофьора: 1–5 звезди + свободен текст (макс. 500 знака).
- Шофьорът оценява пътника: 1–5 звезди + optional текст.
- Рейтингите се показват в профила; средният рейтинг се пресмята от
  последните 50 отзива с exponential decay.
- Не могат да се редактират след 7 дни.

### UC-9: Отмяна и refund
- Пътник отменя >12h преди тръгване → пълен refund (в MVP: mock, без реални
  плащания; статусът е cancelled, местата се освобождават).
- Пътник отменя <12h → mark as no-show; влияе на рейтинга.
- Шофьор отменя → безусловно освобождава пътниците + strike в аудит лог.
- 3 strikes за шофьор в рамките на 90 дни → auto-block, ръчна ревизия.

### UC-10: Админ панел
- Списък от всички users с филтри (роля, статус, дата на регистрация).
- Аудит лог с търсене.
- Статистика: активни маршрути, брой резервации по градове, top шофьори.
- Действия: block/unblock user, force-cancel ride.

## 4. Нефункционални изисквания

| Категория | Изискване |
|---|---|
| **Performance** | Search response ≤ 400ms P95 при 10 000 активни маршрута. |
| **Availability** | 99.5% (планова поддръжка изключена). |
| **Security** | Виж `docs/SECURITY.md` — минимум OWASP ASVS Level 1. |
| **Accessibility** | WCAG 2.1 AA (клавиатурна навигация, aria labels, контраст ≥ 4.5:1). |
| **Локализация** | UI на български (default) и английски; дати по `Europe/Sofia`. |
| **Responsive** | Mobile-first, breakpoints: 480 / 768 / 1024 / 1440. |
| **Browser support** | Последни 2 версии Chrome, Firefox, Safari, Edge. |

## 5. Извън обхвата (не се прави в MVP)

- Реални плащания (Stripe / PayPal) — само mock статуси.
- Мобилни native приложения — само PWA.
- Push нотификации през browser API (в бъдеща фаза).
- Верификация на шофьорска книжка чрез OCR.
- Многоезичен чат превод.
- Интеграция със социални мрежи (Facebook/Google login) — само локален auth.

## 6. Речник

| Термин | Значение |
|---|---|
| **Ride / маршрут** | Обявено пътуване от шофьор с място и час. |
| **Booking / резервация** | Заявка на пътник за N места в конкретен ride. |
| **Seat** | Едно място в кола (не в бекенд entity — просто цяло число). |
| **Rating** | Двупосочна оценка след пътуване. |
| **Audit event** | Неизменяем запис за критично действие в системата. |
